# 學生端「必備功能」10 點建議

> 視角：一位正在申請德國／歐洲大學或碩士班的學生，使用 TaiGer Portal 與留學顧問交流並管理申請進度。
> 目標：讓學生能**直覺掌握進度、清楚知道每間學校的文件要準備什麼、一路到最後送出**，而且不需要任何技術背景就能上手。

本文件針對平台現況（已有 Dashboard、Survey、My Applications、Base Documents、CV/ML/RL Center、Communications 等）提出補強，每一點都說明：**這是什麼 → 為什麼學生需要 → 怎麼讓它更直覺**。

---

## 1. 「一條時間軸」總覽 — 我現在到底在哪一步？

**這是什麼**
一個橫跨整個申請週期的視覺化時間軸／進度條：填資料 → 上傳基本文件 → CV/ML/RL 修改 → 選校定稿 → 各校送件 → 等結果 → 簽證。學生一打開首頁第一眼就看到「我走到第幾步、還剩幾步」。

**為什麼學生需要**
目前 Dashboard 是一堆卡片與任務清單（`StudentDashboard.tsx`、`StudentTaskList.tsx`），任務雖然有優先序，但**沒有「整體旅程走到哪」的全貌**。學生最焦慮的就是「我是不是漏了什麼、是不是落後了」。

**怎麼讓它更直覺**
- 頂部一條 stepper／進度條，每個階段顯示「完成 / 進行中 / 尚未開始」。
- 點任一階段就跳到對應頁面（例如點「文件」直接到 Base Documents）。
- 用一句白話總結現況：「你已完成 60%，下一步：補上推薦信草稿」。

---

## 2. 每間學校的「文件檢查清單」(Per-School Checklist)

**這是什麼**
以**學校 / 學程為單位**的文件待辦清單。每個學程底下列出該校需要的所有文件（CV、ML、各封 RL、成績單、語言證明、APS、Uni-Assist VPD 等），每項都有狀態：未開始 / 草稿中 / 顧問審閱中 / 已定稿 / 已上傳到學校系統。

**為什麼學生需要**
目前文件管理偏「文件導向」(Base Documents + CV/ML/RL Center)，而申請是「學校導向」。學生最常問的就是「**這間學校到底還缺哪些東西？**」每間學校要求不同（有的要 2 封推薦信、有的要動機信限 1 頁），現在很難一眼看出每校缺口。

**怎麼讓它更直覺**
- 每個學程一張卡片，內含一個打勾清單，綠勾 = 完成、灰圈 = 待辦。
- 顯示「3 / 7 份文件就緒」的小進度。
- 直接把文件清單跟 `IApplication`／`document-threads` 綁定，點項目就進到對應的修改 thread。

---

## 3. 截止日期日曆 + 倒數提醒 (Deadline Calendar & Reminders)

**這是什麼**
把所有學程的申請截止日、語言考試效期、APS／Uni-Assist 時程整合成一個**日曆視圖 + 倒數天數**，並在接近期限時主動提醒。

**為什麼學生需要**
這是目前**最關鍵的缺口**：`IApplication` 雖然有 `deadline` 欄位，但沒有日曆視圖、沒有倒數、也沒有任何到期提醒。德國／歐洲申請常常多校、多梯次（冬季/夏季學期），漏掉一個 deadline 等於白做一年。

**怎麼讓它更直覺**
- 月曆 + 列表兩種切換，每個 deadline 標上學校名與剩餘天數。
- 紅／黃／綠色標示緊急程度（< 7 天紅、< 30 天黃）。
- 與第 10 點的通知串接：到期前 14/7/1 天自動提醒。

---

## 4. 主動式通知中心 (Notification Center)

**這是什麼**
一個統一的「收件匣」式通知中心（鈴鐺圖示 + 未讀數），集中所有重要事件：顧問留了新回饋、文件被退回、deadline 將至、面試已排定、錄取結果更新。

**為什麼學生需要**
目前提醒分散在各頁的 banner（`EnglishScoreBelowRequirementBanner` 等），而且一旦關閉就不再出現，缺乏一個「我錯過的事都在這裡」的單一入口。學生不該需要每天逐頁巡邏才知道有沒有新狀況。

**怎麼讓它更直覺**
- 右上角鈴鐺，點開是時間排序的通知列表，每條可一鍵跳到來源頁面。
- 區分「需要我行動」與「純資訊」兩類。
- 搭配 Email／LINE 推播（學生 Profile 已有 LINE ID 欄位），不用一直登入也不會漏。

---

## 5. 顧問回饋與「下一步該做什麼」一目了然

**這是什麼**
在文件修改流程（CV/ML/RL thread）中，清楚標示「**現在輪到誰**」——是等顧問改、還是等學生回？並把「待學生處理」的項目集中顯示。

**為什麼學生需要**
文件來回修改最容易卡在「我以為在等顧問、顧問以為在等我」。系統已有 `latest_message_left_by_id`、`isFinalVersion` 等資料，但學生端沒有把「球在誰手上」講清楚。

**怎麼讓它更直覺**
- 每個 thread 一個明確標籤：`⏳ 等你回覆` / `🔵 顧問審閱中` / `✅ 已定稿`。
- Dashboard 開一個「需要你處理 (N)」區塊，只放等學生行動的事項。
- 定稿時給明確的「這份文件完成了 🎉」回饋，讓學生有完成感。

---

## 6. 引導式新手上路 (Step-by-Step Onboarding Wizard)

**這是什麼**
把目前一頁到底的 Survey（`Survey/index.tsx`）改成分步驟的精靈：基本資料 → 學歷成績 → 語言能力 → 申請偏好 → 上傳第一份文件，每步都有進度與說明。

**為什麼學生需要**
新生第一次進來面對一張超長表單很容易卡住或填不完。Survey 是後續所有任務的前提（很多任務都檢查 `check_academic_background_filled()`），第一步體驗不好，後面全卡住。

**怎麼讓它更直覺**
- 每步只問一小組問題，頂部顯示「第 2 / 5 步」。
- 每個欄位旁加白話 tooltip（例：「GPA 請填你成績單上的總平均，不確定可先跳過」）。
- 可隨時存檔離開、之後接續，不強迫一次填完。

---

## 7. 選校清單與排序 (My School List / Shortlist)

**這是什麼**
讓學生能把有興趣的學程**加入我的清單、收藏、排序**（夢幻校 / 主力校 / 保底校），並標記每間的決定狀態。

**為什麼學生需要**
平台有 Program 瀏覽與比較（`ProgramCompare.tsx`），但沒有「收藏 / 我的口袋名單」。學生在和顧問討論選校時，需要一個自己能整理、拖拉排序、加註想法的地方，而不是只在聊天裡口頭講。

**怎麼讓它更直覺**
- 程式列表上一顆「⭐ 收藏」按鈕。
- 我的清單頁可拖拉排序、分「衝刺/主力/保底」三層。
- 每間可加一句私人筆記（「喜歡這間的 AI 實驗室」），方便和顧問討論。

---

## 8. 文件需求說明與範例庫 (Guidance & Examples)

**這是什麼**
針對每種文件（CV / 動機信 ML / 推薦信 RL / 成績單 / APS / 語言證明）提供白話說明、格式要求、常見錯誤、以及（去識別化的）範例或範本。

**為什麼學生需要**
題目核心是「**知道要如何準備各個學校文件**」。第一次申請的學生根本不知道德式動機信和英美 SOP 差別、APS 是什麼、Uni-Assist 怎麼運作。現有 Learning Resources 仍是「Coming soon」的占位頁。

**怎麼讓它更直覺**
- 在每個文件 thread／清單項目旁放「📖 怎麼準備這份？」連結。
- 內容用問答式短段落 + 範本下載，避免長篇大論。
- 針對德國／歐洲特有流程（APS、Uni-Assist、學期制）做專章。

---

## 9. 截止前自動提醒與摘要（Email / LINE 推播 + 每週摘要）

**這是什麼**
把第 3、4 點的事件透過學生熟悉的管道（Email、LINE）主動送出：deadline 倒數提醒、顧問新回饋通知、每週一封「本週待辦摘要」。

**為什麼學生需要**
學生不會每天登入平台。真正能避免「漏送件、漏回覆顧問」的，是把訊息送到他每天會看的地方。Profile 已收集 Email 與 LINE ID，基礎資料已具備。

**怎麼讓它更直覺**
- 提供通知偏好設定：哪些事要推、用 Email 還是 LINE、頻率。
- 每週摘要用三句話講完：「本週 2 個 deadline、1 份文件等你回覆、上週完成 3 項」。
- 每封通知都附一個直達連結，一點就回到該做的事，零摸索。

---

## 總結：這 9 點如何串成一條順暢體驗

| # | 功能 | 解決的學生痛點 |
|---|------|----------------|
| 1 | 申請時間軸總覽 | 「我走到哪了？」 |
| 2 | 每校文件檢查清單 | 「這間還缺什麼？」 |
| 3 | 截止日曆 + 倒數 | 「什麼時候要交？」 |
| 4 | 通知中心 | 「我有沒有漏掉事情？」 |
| 5 | 球在誰手上 + 下一步 | 「現在換我做嗎？」 |
| 6 | 引導式上手精靈 | 「第一次進來不知從何開始」 |
| 7 | 選校口袋名單 | 「我到底要申哪幾間？」 |
| 8 | 文件說明與範例 | 「這份文件要怎麼寫？」 |
| 9 | 自動提醒與每週摘要 | 「不用一直登入也不漏事」 |

**設計原則貫穿全部**：少打字多打勾、用白話不用術語、永遠告訴學生「下一步做什麼」、把訊息送到學生本來就在的地方。讓沒有任何技術背景的學生，也能安心地一路走到送出。
